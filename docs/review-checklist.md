# 리뷰 체크리스트

## 범위

- [ ] 요청된 기능만 변경했다.
- [ ] 관련 없는 리팩터링과 설정 변경이 없다.
- [ ] 가정과 남은 위험을 기록했다.

## API

- [ ] URI, Method, 요청, 응답이 `docs/API_SPEC.md`와 일치한다.
- [ ] 필드 의미와 자료형이 `docs/API_FIELD_GUIDE.md`와 일치한다.
- [ ] API 변경 시 명세, 컬럼 설명, 변경 이력을 함께 수정했다.
- [ ] 1차 스프린트에 인증, 편집 토큰, `version`을 임의로 추가하지 않았다.

## Spring

- [ ] Controller, Service, Repository, Client의 책임이 분리됐다.
- [ ] DTO와 Entity를 분리했다.
- [ ] 외부 API DTO와 내부 도메인 모델을 분리했다.
- [ ] 트랜잭션 경계가 Service에 있다.

## DB

- [ ] Entity와 컬럼이 `docs/ERD.md`와 일치한다.
- [ ] Entity 변경 시 ERD를 함께 수정했다.
- [ ] migration 도구를 사용 중이면 새 migration을 추가했다.
- [ ] 공유된 기존 migration을 수정하지 않았다.
- [ ] N+1과 고유 조건을 검토했다.

## 테스트와 보안

- [ ] 정상 흐름과 주요 실패 흐름을 테스트했다.
- [ ] 외부 API 장애와 누락 응답을 처리했다.
- [ ] 실행한 테스트와 결과를 기록했다.
- [ ] API Key와 비밀값이 코드, 문서, 로그에 없다.
- [ ] 공개 오류에 내부 예외와 스택 트레이스가 없다.
